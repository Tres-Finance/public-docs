## api

> Tres GraphQL API reference — endpoints by domain with full example queries

- [address-book](api/address-book.md): Manage custom labels/tags for external (counterparty) addresses and query the address book with per-account inflow/outflow transaction summaries, including unidentified addresses. Endpoints: Set Custom Account Name; Set Tags to Custom Account Name; Get Custom Account Name Tags; Delete Custom Account Name; Get Account Tx Summary; Get Address Book with Tx Summary; Get Address Book without Tx Summary; Get Unidentified Addresses
- [admin](api/admin.md): Org administration: locked accounting periods (CRUD), user invites/roles/deletion, organization and platform settings, ledger task inspection, and skipping/disabling data collection per platform. Endpoints: Get Locked Period History; Create Locked Period; Update Locked Period; Delete Locked Period; Invite Users; Get Users List; Delete Invite; Delete User; Get Admin Page / Settings; Set User Type; Set Organization Settings; Get Platform Settings (+3 more)
- [agent](api/agent.md): Tres AI agent chat API: start a session, send messages, and fetch conversation history by session ID. Endpoints: Get Agent History; Start Agent Session; Send Message To Agent
- [anyv](api/anyv.md): AnyV AI-powered CSV transaction import pipeline: generate upload URL, map columns, surface mapping errors/insights/unique examples, start/continue processing, and dismiss/cancel CSV workflows. Endpoints: DismissCsvNotification; CancelCsvWorkflowMutation; AiDismissTxsCsv; AiGenerateUploadTXCSVUrlMutation; AiGetCSVMappingErrorsMutation; AiGetCSVInsightsMutation; AiGetCSVUniqueExamplesMutation; AiStartProcessingTXSCSVMutation; AiContinueCsvToProcessingMutation; AiUploadCsvMappingMutation
- [asset-balances--archived-portfolio-tracking](api/asset-balances--archived-portfolio-tracking.md): Historical portfolio tracking via the `portfolio` query: daily net-worth time series over a date range — total, per asset class/underlying asset, per internal account, and per balance. Distinct from the statefull part (current committed balances) and the stateless part (live on-chain reads). Endpoints: Get Daily Total Networth; Get Daily Networth Per Asset Class / Underlying Assets; Get Daily Networth Per Internal Account; Get Daily Networth Per Balance
- [asset-balances--statefull](api/asset-balances--statefull.md): The main committed (DB-state) balance queries: org-level balances and positions, `assetBalance` root/underlying balances with cost basis, counter-transfer state, claimables, asset metadata, commit group times, archived balances, and manual wallet balance updates. Use this part for balances as recorded by the last commit — not live on-chain reads (see --stateless) or daily net-worth history (see --archived-portfolio-tracking). Endpoints: Get Organization Positions; Get Organization Per Wallet; Get Organization Balance; Get Organization Balance of DeFi Assets; Get Organization Balance + Cost Basis; Get Organization Balance + Counter Transfers State; Get Asset Balances (Root Balances); Get Asset Balances (Underlying Balances); Get Asset Balances with Realtime (Stateless) Data; Get Asset Balances by Activity Time; Get Organization Balance Fiat Value; Get Claimables (+8 more)
- [asset-balances--stateless](api/asset-balances--stateless.md): Realtime on-chain reads (no commit needed): stateless DeFi/staking positions and token balances for arbitrary wallet identifiers at an optional timestamp/block, plus the list of supported platforms. Use this part for live/ad-hoc balance lookups; committed DB balances live in --statefull. Endpoints: Stateless DeFi / Staking Positions; Stateless Token Balance; Supported Stateless Balances & Positions
- [classification](api/classification.md): Transaction classification rules CRUD (activity/balance-factor rules matched by method ID, platform, sender/recipient identifiers and tags) plus listing classification workflows/tasks. Endpoints: Get Classification Rules; Get All Workflows / Tasks; Create Classification Rule; Update Classification Rule; Delete Classification Rule
- [commit-data-collection-managment](api/commit-data-collection-managment.md): Data-collection commit management via `collectAudit`: query last/last-successful/running commit status by commit ID, trigger commits (org-wide, per internal account, with time range or commit ID), parallel commits with max-transaction limits, reconciliation checks, and killing commits. Endpoints: Get Last Success Commit; Get Last Success or Running Commit; Get Commit Status by Commit ID; Get Last Commit; Trigger Commit; Trigger Commit By Internal Account; Trigger Commit By Internal Account + Time; Trigger Commit By Internal Account + Time + Commit ID; Trigger Parallel Commit; Trigger Parallel Commit + Max Transaction Limit; Update Parallel Commit Max Transaction Limit; Check Parallel Commit Reconciliation (+2 more)
- [cost-basis](api/cost-basis.md): Cost-basis management: asset reevaluations/impairments (CRUD), Spec ID rules (set/delete/query by sub-tx or rule ID), triggering cost-basis calculation, inventory configuration, per-date strategy updates, and calculation status. Endpoints: create reevaluation; update reevaluation; delete reevaluation; get reevaluations; set reevaluation internal account; Set Spec ID Rule; Delete Spec ID Rule; Get Spec ID Rules; Get Spec ID Rules By SubTx ID; Get Spec ID Rules By Rule ID; Trigger Cost Basis; Set Full Cost Basis Inventory Configuration (+3 more)
- [custodian-sync](api/custodian-sync.md): Create a custodian integration (Anchorage example) via `createCustodianIntegration`, configured either for Proof of Funds + FinOS or PoF only. Endpoints: Integrate Anchorage Custodian [PoF + FinOS]; Integrate Anchorage Custodian [Only PoF]
- [favorite-ledger-view](api/favorite-ledger-view.md): Saved ledger filter presets: save the full transaction-filter parameter set as a named favorite view, then list, fetch by name, rename, or delete it. Endpoints: Save favorite ledger view; Get Favorite ledger views; Get Favorite ledger view by name; Update name for favorite ledger view; Delete favorite ledger view
- [form-filler](api/form-filler.md): AI form-filling service: submit form inputs to have fields filled with AI reasoning per field, and send feedback on results. Endpoints: FormFiler - Fill Form(s) with AI; FormFiler - Send Feedback
- [integrations--actions-1](api/integrations--actions-1.md): ERP integration mutations (bulk of the actions): NetSuite auth setup, syncing/unsyncing transactions to the ERP (all, specific, bulk, by entity source type), ignore/undo-ignore, creating/updating ERP mapping rules of every type (custom, default, internal transfer, specific sub-tx, ignore), manual ERP creation, logout/revoke, and integration-account upsert/delete. Endpoints: Create or Update Netsuite AUth; Sync All; Sync Specific Txs; Sync Specific Txs With Entity Source Type; Ignore Tx; Undo Ignore Transaction; Unsync Specific Txs; Unsync Specific Txs With Entity Source Type; Create or Update Rule - Custom Rule; Create or Update Rule - Default; Create or Update Rule - Internal Transfer; Create or Update Rule - Specific Sub Transaction (+8 more)

All other integration mutations are in --actions-1. Endpoints: Unexclude SubTransactions From Rule; exclude SubTransactions From Rule Copy
- [integrations--queries](api/integrations--queries.md): ERP integration read side: transactions with their ERP rules/journal entries, rule listings (custom/default/internal-transfer), AP/AR, invoices, bills, chart of accounts, related organizations, integration status, tx/sub-tx syncing status, and connect links for QBO/Slack/Deel. Mutations live in --actions-1/--actions-2. Endpoints: Get transaction with subtxs + erp rules; Get Rules - Custom Rules & Default & netsuite data; Get Rules - Internal Transfer Rule; Get APAR; Get Invoices; Get Bills; Get ERP Chart of Accounts; Get related organizations; Get ERP Integration; Get ERP TX Syncing Status; Get QBO Link; Get Slack Link (+6 more)
- [internal-account-wallets](api/internal-account-wallets.md): Internal account (tracked wallet) lifecycle: query accounts by identifier/status/asset class with balances and fiat values, batch create/update (on-chain and exchange/offchain), update name/description/status/tags, delete, and clean account data. Endpoints: Get Internal accounts by Identifier; Get Internal accounts by Statuses; Get Internal Account Full List and Status; Get Internal Accounts by Asset Class; Create / Update Batch Internal Accounts; Create / Update Batch Internal Accounts (Exchange / Offchain); Update Internal Account Name / Description; Update Internal Account Status; Get Internal Account By ID; Set Internal Account Tags; Delete Internal Account; Clean Internal Account (+2 more)
